# 协作规则

## 基本原则

1. **各自优势**：发挥每个助理的独特能力
2. **知识共享**：有价值的经验及时同步到共享知识库
3. **协调分工**：复杂任务分工合作，避免重复劳动

## 分工原则

| 场景 | 推荐助理 |
|------|---------|
| 需要稳定网络/快速响应 | Max（云端） |
| 本地文件操作/私密任务 | H（本地） |
| 复杂工作流/调试任务 | Eva（本地） |

## 同步机制

### 技能同步

- 各助理在 `skills/{name}/` 维护专属技能
- 共享技能放在 `skills/shared/`
- 每次重要迭代后推送更新

### 记忆同步

- 重要上下文存入 `memory/`
- 用户偏好和规则存入 `memory/rules/`
- 协作记录存入 `memory/context/`

### 冲突处理

1. 优先保留更新版本（检查 git log）
2. 冲突时保留双方内容，标记待讨论
3. 重大冲突通知用户决策

## 通信约定

- 通过 GitHub 作为中转站
- 每次 commit 需注明：谁、做了什么、日期
- 重大变更前在 commit message 说明

## 安全注意

- 禁止同步包含密钥的文件（如 auth.json, .env）
- 敏感信息只记录加密后的哈希值
- 定期清理临时文件
